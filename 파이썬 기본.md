> 💡 토막 상식: 지연 평가와 반복 객체<br>

    파이썬의 메모리 최적화와 동작 원리를 이해하기 위한 핵심 개념입니다.<br>
    ① 지연 평가 (Lazy Evaluation)란?
    모든 값을 한 번에 메모리에 올리지 않고, 필요할 때만 즉석에서 값을 만들어내는 특성입니다.

<br><br>
② 이터러블 (Iterable)
반복 가능한 객체 그 자체를 말합니다. 내부 요소를 하나씩 반환할 수 있는 상태입니다.
<br>
**종류**: list, tuple, str, dict, range() 등
<br>
**특징**: iter() 함수를 사용해 이터레이터로 변환할 수 있습니다.
<br><br>
③ 이터레이터 (Iterator)
반복 가능한 객체(Iterable)에서 값을 하나씩 꺼낼 수 있는 상태의 객체입니다. map(함수, 이터러블)을 실행하면 그 결과를 이터레이터로 반환합니다.
<br>
**특징**: next() 함수를 통해 다음 값을 순차적으로 꺼낼 수 있습니다.
<br>
비유하자면:<br>
이터러블: 책 (전체 내용을 담고 있음)<br>
이터레이터: 책갈피 (한 페이지씩 넘겨줌)<br>
<br>
④ 제너레이터 (Generator)
이터레이터를 아주 쉽게 만들어주는 특별한 도구입니다.<br>
함수형 방식: 일반적인 return 대신 yield를 여러 개 사용하여 값을 순차적으로 반환합니다.
<br>
표현식 방식: 리스트 컴프리헨션과 비슷하지만 대괄호 [] 대신 소괄호 ()를 사용합니다.

##### 4. 문자열 제거 및 정제 (strip)

    문자열 양끝의 공백이나 특정 문자를 제거할 때 사용합니다. 기본값은 공백 제거이며, 매개변수에 특정 문자열 집합을 넣으면 해당 문자들을 모두 찾아 지워줍니다.

<br>
	```

    Python
    ```

### 3. 리스트 & 리스트 컴프리헨션

#### [Python 기초] 3. 자료구조 (리스트, 튜플, 딕셔너리, 세트)

##### 1. 4대 파이썬 컬렉션(Collection) 요약

무언가를 담는 연속 자료구조 4가지의 특징 비교입니다. 코테에서는 목적에 맞는 자료구조를 선택하는 것이 핵심입니다.

| 자료구조           | 형태        | 순서  |    중복    | 변경  | 비고                                 |
| :----------------- | :---------- | :---: | :--------: | :---: | :----------------------------------- |
| **리스트(List)**   | `[1, 2, 3]` |   O   |     O      |   O   | 배열, 스택, 큐의 기본 형태           |
| **튜플(Tuple)**    | `(1, 2, 3)` |   O   |     O      | **X** | 값이 변하면 안 될 때 사용            |
| **딕셔너리(Dict)** | `{"a": 1}`  |  O\*  | 키 X, 값 O |   O   | 키(Key)로 값(Value)을 빠르게 찾을 때 |
| **세트(Set)**      | `{1, 2, 3}` | **X** |   **X**    |   O   | 교집합/차집합, 중복 제거용           |

> _참고: 파이썬 3.7 버전부터 딕셔너리는 데이터가 추가된 순서를 유지합니다._

##### 2. 리스트(List)의 활용과 시간 복잡도

리스트 하나로 배열, 스택(Stack), 큐(Queue)의 역할을 모두 수행할 수 있지만, 동작에 따라 속도(시간 복잡도) 차이가 큽니다.

```
# 1. 빠른 연산 (O(1)) - 스택처럼 쓰기
arr.append(5)  # 맨 뒤에 추가 (빠름)
arr.pop()      # 맨 끝 요소 꺼내기 (빠름)
```

```
# 2. 느린 연산 (O(N)) - 주의!
arr.insert(0, 1) # 맨 앞이나 중간에 넣기 (뒤에 있는 요소들을 다 밀어야 해서 느림)
arr.pop(0)       # 맨 앞 요소 꺼내기 (마찬가지로 다 당겨와야 해서 느림)
```

```
# 3. 정렬 메서드 비교
arr.sort()       # 원본 리스트 자체를 정렬하여 변경 (반환값 없음)
sorted(arr)      # 원본은 그대로 두고, 정렬된 '새로운 리스트'를 반환
```

##### 3. 큐(Queue)는 무조건 deque를 쓰자

리스트의 pop(0)은 너무 느리기 때문에, 코테에서 큐(Queue)나 BFS 알고리즘을 구현할 때는 반드시 collections 모듈의 deque를 사용해야 합니다.

`from collections import deque`

```
queue = deque([1, 2, 3])
queue.append(4)         # 뒤에 넣기
print(queue.popleft())  # [중요] 앞에서 빼기 (pop(0) 대신 사용, 속도 빠름!)
# 출력: 1
```

##### 4. 리스트 컴프리헨션 (List Comprehension)

반복문과 조건문을 한 줄로 줄여서 리스트를 생성하는 파이썬의 강력한 문법입니다.

```
# 일반적인 for문 사용
arr = []
for i in range(10):
    if i % 2 == 0:
        arr.append(i)

# 리스트 컴프리헨션 (위 코드를 한 줄로!)
arr = [i for i in range(10) if i % 2 == 0]
```

##### 5. 탐색, 카운팅, 중복 제거 (Dict & Set)

##### 6. 코테 치트키: defaultdict

딕셔너리에 없는 키를 조회할 때 발생하는 KeyError를 방지해 주는 아주 유용한 모듈입니다.

`from collections import defaultdict`

```
# 값이 없으면 자동으로 정수 기본값(0)으로 초기화해주는 딕셔너리 생성
d = defaultdict(int)
words = ['book', 'book', 'pen']

for w in words:
    d[w] += 1  # 원래라면 d[w]가 없어서 에러가 나지만, defaultdict라 0에서 1을 더해줌!

print(dict(d))
# 출력: {'book': 2, 'pen': 1}
```
